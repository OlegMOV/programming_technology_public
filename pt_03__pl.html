2. Введення / виведення даних у консоль

Теорія - 15 хв, практика - 0 хв

Для введення даних для використання у виразах набраних у консолі, є відповідна службова інструкція input(). Для виведення даних у консоль використовують print().
Для використання таких інструкцій доцільно використовувати змінні. Що забезпечить написання більш зручного для читання коду. Але можливо і використовувати вирази прямо в дужках даних інструкцій.

Для уточнення інформації щодо інструкцій в консолі можливо викликати інструкцію help(arg), де arg - інструкція щодо якої потрібна інформація, наприклад:
>>> help(print)
Help on built-in function print in module builtins:
>print(*args, sep=' ', end='\n', file=None, flush=False)
    Prints the values to a stream, or to sys.stdout by default.
    sep
      string inserted between values, default a space.
    end
      string appended after the last value, default a newline.
    file
      a file-like object (stream); defaults to the current sys.stdout.
    flush
      whether to forcibly flush the stream.

4. Документування коду програми

Теорія - 15 хв, практика - 0 хв

Загальні вимоги до коду програми:

  • код легко читати і сприймати;
  • код легко змінювати;
  • код можна повторно використати і тестувати;
  • код використовує безкоштовні платформу і ПЗ;
  • мінімальне використання ресурсів;
  • стабільність при зміні конфігурації / середовища.
    Вище розглянуто було важливість і правила щодо вибору назв змінних. Для покращення читабельності коду важливим є застосовування коментарів до частин коду.
Коментар має описувати мету, яку маємо досягти цією частиною коду, а не алгоритм її досягнення.

Потрібно описувати навіщо написаний даний фрагмент коду, оскільки яким чином досягнутий результат може бути реалізовано багатьма способами.
