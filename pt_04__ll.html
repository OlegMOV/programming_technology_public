НАВЧАЛЬНІ ПИТАННЯ

1. Цикл по колекції

Теорія - 20 хв, практика - 0 хв

Розглянувши на попередніх заняттях код лінійної програми можливо дійти висновків, що послідовне виконання може мати випадок коли одні і ту саму послідовність інструкцій необхідно повторити кілька раз. Реалізувати це можна шляхом повторення відповідного коду необхідну кількість раз у тексті програми. Але тут є кілька обмежень. По-перше, це ускладнює читабельність коду і його збільшення. По-друге, необхідна кількість повторень, як правило, визначається вже у ході виконання програми.

Прикладом може бути код програми для введення особистих даних особового складу підрозділу. Для цього необхідно реалізувати одні і ті ж інструкції для кожного окремого військовослужбовця. І це достатньо поширена практика, тому у будь якій мові програмування є механізм, який дозволяє реалізувати логіку повторення блоків інструкцій.

Оператори, які дозволяють реалізувати це, називають циклічними операторами. Тобто, циклічні оператори реалізують конструкцію повторення, яку ще називають циклом. Кожне окреме повторення - крок циклу або ітерація.

Всі цикли складаються з заголовку та тіла циклу. Заголовок циклу описує умову повторення циклу. Тіло циклу - містить у собі блок інструкцій, які реалізують логіку одного повторення (ітерації).

Для більш гнучкої реалізації повторень блоку інструкцій у мові програмування Python застосовується дві конструкції циклічних операторів:

Рис. 1.1

**Цикл по колекції.** Синтаксис такого оператора наступний: ```python for variable in sequence: < instructions block > ``` - `sequence` - послідовність, що містить у собі елементи, які, як правило, використовуються у тілі циклічного оператора; - `variable` - змінна, яка приймає послідовно значення елементів послідовності `sequence`; - `instructions block` - блок з інструкціями, які реалізують логіку ітерації / кроку циклу. У подальшому буде розглядатися така структура даних як колекція, яка по суті є послідовність (`sequence`) змінних одного чи більше типів. На даний момент послідовність будемо розуміти як сукупність фіксованої кількості значень будь-якого типу.
Змінна строкового типу є також послідовністю, елементами якої є усі символи рядка, включно із символами пробілів. Наприклад:
line = 'Military institute'
for char in line:
    print(char, end='')
# Вивід у консоль буде
#  Military institute

Циклічний оператор забезпечує послідовний доступ до кожного елементу. Тому, як приклад, розглянемо послідовність чисел

list_number = [1, 2, 3, 4]
for var in list_number:
    print("| ", var, " |")

За результатами виконання зазначеного коду буде виведена наступна послідовність символів у консолі:

|  1  |
|  2  |
|  3  |
|  4  |

У зазначеному прикладі елементи послідовності присвоюються по черзі змінній var на кожній ітерації. У подальшому виконується тіло циклу, у якому є можливість використати поточне значення змінної var. На наступній ітерації значення змінної посилається на наступне значення послідовності.

Змінна var дійсна і поза межами тіла циклу. Її значення рівне значенню на момент закінчення ітерацій циклу
Використання змінної із заголовку циклічного оператора за межами тіла циклу є поганою практикою, оскільки знижує читабельність коду програми

Слід звернути увагу, що у циклічному оператору послідовність введена як змінна list_number. Тобто, кількість елементів послідовності може бути різною на момент виконання даного фрагменту коду. Хоча і послідовність може бути записана і конструкції циклічного оператора:

for var in [1, 2, 3, 4]:
    print("| ", var, " |")

Послідовність чисел, що збільшуються / зменшуються на дискретне значення є достатньо частотним по використанню елементом у програмуванні. Тому для цього є вбудована функція range(). Для її використання необхідно задати параметри, які визначають розмір та значення такої послідовності.

  1. При введені одного параметру range(stop) – задання послідовності цілих чисел від 0 до stop-1 з кроком 1.
  2. Повний запис вбудованої функції наступний:
    range(start, stop[, step]) - задання послідовності (арифметичної прогресії) від start до stop-1 з кроком step.
    Якщо параметр step опущений, він за замовчуванням дорівнює 1.
    Приклади:
  • range(5) отримаємо діапазон в з елементів 0, 1, 2, 3, 4
  • range(1, 5) отримаємо діапазон в з елементів 1, 2, 3, 4
  • range(0, 10, 3) отримаємо діапазон в з елементів 0, 3, 6, 9

Для реалізації зворотного відліку використовують наступну конфігурацію аргументів функції range:

for var in range(7,0,-1):
   print("| ", var, " |")

За результатами виконання зазначеного коду буде виведена наступна послідовність символів у консолі:

|  7  |
|  6  |
|  5  |
|  4  |
|  3  |
|  2  |
|  1  |
Аргумент функції range(), що визначає кінцевий елемент послідовності, в послідовність НЕ ВКЛЮЧАЄТЬСЯ

Циклічний оператор може бути вкладеним у блок інструкцій іншого циклічного оператора. Так наприклад, якщо є необхідність перебору елементів матриці, двовимірної послідовності, застосовують наступну комбінацію інструкцій:

for i in range(3):
    for j in range(2):
        print('M[',i, ', ', j, ']')

За результатами виконання зазначеного коду буде виведена наступна послідовність символів у консолі:

M[ 0 ,  0 ]
M[ 0 ,  1 ]
M[ 1 ,  0 ]
M[ 1 ,  1 ]
M[ 2 ,  0 ]
M[ 2 ,  1 ]

Тобто, аналізуючи результати виведення, послідовність виконання зазначеного коду наступна:

  1. Присвоєння змінній i початкового значення послідовності range(3).
  2. Послідовний перебір послідовності range(2) у значенні змінної j.
  3. Присвоєння змінній i наступного значення послідовності range(3).
  4. Виконання пункту 2.
  5. І так далі, доки i не набуде значення 2, а j = 1.
  6. Закінчення виконання конструкції із двох циклічних операторів.

Висновки:

  • застосування циклічного оператора дозволяє уникнути дублювання коду у тексті програми для реалізації ідентичних дій;
  • у тіло циклічного оператора можуть бути вкладені інші циклічні оператори, що дозволяє реалізовувати гнучкий доступ до багатовимірних структур даних.

2. Цикл з умовою продовження

Теорія - 15 хв, практика - 0 хв

Розглянутий цикл по колекції дозволяє реалізувати повторення тіла циклу (блоку інструкцій) визначену, на момент виконання, кількість разів. Але є багато ситуацій коли при розробці алгоритму програми кількість ітерацій може змінюватися у процесі виконання інструкцій тіла циклу.
Наприклад, у процесі введення даних щодо особового складу, який згадували раніше, можна передбачити переривання циклу за введенням строки відповідних символів. Оскільки підрозділ може бути достатньо великим, і оператору потрібно зробити перерву на відпочинок, при цьому не втративши введені дані. Переривання циклу за умовою є достатньо частотним завданням. Тому у Python є відповідний оператор, що реалізує таку функціональність - циклічний оператор з умовою продовження. Синтаксис його запису наступний:

while умова:
    тіло циклу

Як і попереднього оператора, даний оператор складається із заголовку та тіла циклу.
Логіку роботи оператора можна представити наступним чином:

Рис. 2.1

Що таке умова та синтаксис її написання буде розглянуто нижче. Поки будемо розуміти, що це конструкція, за результатами виконання якої повертається змінна типу bool (True або False).
Для прикладу розглянемо наступний фрагмент коду програми:

# Чисельність підрозділу
amount = 30
# Порядковий номер
count = 1
while count < amount:
    data = input('Прізвище особи: ')
    count += 1
# запис введених даних до бази даних

У даному прикладі умова у заголовку циклу порівнює значення змінної count із значенням змінної amount. За умови вірності виразу виконується тіло циклу, в іншому випадку тіло циклу не виконується, а далі виконуються інструкції, що записані після блоку запису оператора while.
З використанням оператора while можна реалізувати функціональність оператора for, наприклад у випадку перебору послідовності чисел:

print("Результати виконання оператора for")
for number in range(3):
    print("| ", number, " |")
# 
print("Результати виконання оператора while")
number = 0
while number < 3:
    print("| ", number, " |")
    number += 1

За результатами виконання зазначеного коду буде виведена наступна послідовність символів у консолі:

Результати виконання оператора for
|  0  |
|  1  |
|  2  |
Результати виконання оператора while
|  0  |
|  1  |
|  2  |
Важливо слідкувати за тим щоб умова закінчення виконання циклу була можливою. Інакше цикл не буде мати закінчення, і, відповідно, код програми продовження (безконечний цикл).

Такі помилки трапляються достатньо часто, наприклад у вищезазначеному прикладі рядок number += 1 у тілі циклу є важливим. Оскільки тільки він відповідає за реалізацію умови закінчення циклу. У разі його відсутності цикл стає безконечним.
Циклічний оператор з умовою продовження може бути вкладеним у циклічні оператори та інші оператори управління кодом.

Висновки:

  • циклічний оператор з умовою продовження дозволяє виконувати тіло циклу у випадку коли умова у заголовку циклу має значення True. У випадку коли значення умови рівне False, тіло циклу взагалі не виконується. Продовжується виконання інструкцій основної програми, які йдуть після циклічного оператора;
  • кількість ітерацій циклічного оператора з умовою продовження залежить від значення змінних, які є складовими виразу, що описує умову у заголовку циклічного оператора.

3. Переривання та продовження циклів

Теорія - 15 хв, практика - 0 хв

Для можливості більш гнучкого управління програмою, а саме під час виконання тіла циклічних операторів, у Python є інструкції переривання ітерацій.
Інструкція break, що записана у тілі циклічного оператору, негайно перериває подальше виконання ітерацій, та ініціює вихід із циклічного оператора.
Для прикладу наступний фрагмент коду:

print('Початок виконання циклічного оператора')
for var in range(3):
    print(var)
    break
print('Закінчення виконання циклічного оператора')

За результатами виконання зазначеного коду буде виведена наступна послідовність символів у консолі:

Початок виконання циклічного оператора
0
Закінчення виконання циклічного оператора

У даному випадку виконується одна ітерація, що виводить у консоль значення змінної var, яке рівне 0. І у подальшому інструкція break перериває виконання подальших ітерацій. Після виходу з циклічного оператора продовжується виконання інструкцій коду програми.
Таким же чином інструкція break діє і у циклічному операторі while.
Крім того, є інструкція continue, функціональність якої схожа із попередньою, але дана інструкція перериває виконання поточної ітерації. Тобто після її спрацювання виконання подальших інструкцій тіла циклу переривається, і виконання коду переходить до початку наступної ітерації. 

var = 0
print('Початок виконання циклічного оператора')
while var < 2:
    print('Початок ітерації ', var+1)
    print(var)
    var += 1
    continue
    print('Закінчення ітерації ', var+1)
print('Закінчення виконання циклічного оператора')

За результатами виконання зазначеного коду буде виведена наступна послідовність символів у консолі:

Початок виконання циклічного оператора
Початок ітерації 1
0
Початок ітерації 2
1
Закінчення виконання циклічного оператора

Проаналізуємо логіку роботи фрагменту коду.

  1. Присвоєння змінній var значення 0, та вивід у консоль рядку, введеного у операторі print().
  2. Початок виконання циклічного оператора. Перевірка умови var < 2.
  3. Виконання першої інструкції print() із тіла циклу.
  4. Виконання другої інструкції print() із тіла циклу.
  5. Збільшення змінної var на одиницю.
  6. Інструкція continue перериває подальше виконання тіла циклу.
  7. Перевірка умови var < 2.
  8. Виконання першої інструкції print() із тіла циклу.
  9. Виконання другої інструкції print() із тіла циклу.
  10. Збільшення змінної var на одиницю.
  11. Інструкція continue перериває подальше виконання тіла циклу.
  12. Перевірка умови var < 2.
  13. Вихід із циклічного оператора.
  14. Вивід у консоль рядку, введеного у інструкції print(), що іде за конструкцією циклічного оператора.

На практиці інструкції переривання використовуються з умовними операторами, які будуть розглядатися на наступному занятті.
Для прикладу, реалізуємо код, який здійснює пошук елемента послідовності, що відповідає заданій умові i == 11. Після знаходження такого, подальше виконання тіла циклічного втрачає необхідність.

numbers = [32, 5, 45, 11, 10, 12]
for i in numbers:
    if i == 11:
        print("Знайдено 11!")
        break

Тобто на четвертій ітерації буде знайдено число 11, і далі здійснюється переривання виконання тіла циклічного оператору.
Для забезпечення можливості більш гнучкої написання логіки саме циклічних операторів програми, їх конструкція була доповнена ще однієї інструкцією else:. Суть якої достатньо проста, блок операторів у даній інструкції буде виконаний лише при умові, що у тілі циклічного оператору не була застосована інструкція break. Для прикладу доповнимо попередній фрагмент коду:

numbers = [32, 5, 45, 11, 10, 12]
for i in numbers:
    if i == 11:
        print("Знайдено 11!")
        break
else:
    print('Число 11 не знайдено. :(')

У зазначеному фрагменті коду не буде виконаний блок інструкцій оператора else:, оскільки було застосовано переривання. При умові, наприклад if i == 24 - інструкції блоку else: будуть виконані.

Інструкція "else:" є не обов'язковою при записі циклічних операторів. Використовується лише за необхідності розширення функціоналу

Висновки:

  1. Інструкції переривання циклічних операторів забезпечують можливість здійснювати управління потоком інструкцій у тілі циклу.
  2. Інструкція break дозволяє перервати виконання тіла циклу не зважаючи на те що умова виходу із циклу while ще не виконана, а для оператора for не закінчився перебір змінних у послідовності заголовку циклу.

4. Основи алгебри висловлювань

Теорія - 15 хв, практика - 0 хв

У житті досить часто ми приймаємо рішення, які буквально інтерпретуються як так чи ні. Або отримуємо чи задаємо питання, які потребують схвальної чи не схвальної відповіді. Більш формально поставлені питання називають висловлюваннями.

Отже, висловлювання - твердження, яке може мати одне з двох булевих значень True або False.
При розробці програми, висловлювання використовуються доволі часто. Оскільки це є необхідна конструкція для реалізації логіки алгоритму, який передбачає перевірку відповідності значень змінних визначеним критеріям. Як правило, висловлювання починає діяти на момент коли виконання програми підійшло до місця де необхідно прийняти рішення подальшого варіанту виконання коду. Для цього записується висловлювання, яке дозволяє порівняти поточне значення змінної із заданим критерієм. Наприклад, amount < 8, може True чи False в залежності від значення змінної amount на момент виконання. Результатом висловлювань є значення типу bool.
У Python у висловлюваннях для об'єктів (змінних різних типів) є наступні операції порівняння

Таблиця 4.1 Оператори порівняння

Оператор Значення
> більше;
< менше;
>= більше або рівне (не менше);
<= менше або рівне (не більше);
== дорівнює (рівне);
!= не дорівнює (не рівне).

Розглянуті операції застосовують наступним чином:

>>> x = 5
>>> x < 7
True
>>> x > 5
False
>>> x == '5'
False

У мові програмування Python крім висловлювань може приймати булеве значення і об'єкт. Наприклад, якщо число відрізняється від нуля - його логічне значення - True, навпаки - False.

>>> counter = 0
>>> counter == True
False

Для можливості складання більш складних логічних виразів над висловлюваннями визначено три базових логічних (булевих) операції:

  • Диз'юнкція (логічне «або» позначається or) – це бінарна операція, що діє за правилом: висловлювання p or q хибне тоді і тільки тоді, коли хибні одночасно висловлювання p і q, та істинне в інших випадках.
  • Кон'юнкція (логічне «і» позначається and) – це бінарна операція, що діє за правилом: висловлювання p and q істинне тоді і тільки тоді, коли істинні одночасно висловлювання p і q, та хибне в інших випадках.
  • Заперечення (логічне «не» позначається not) – це унарна операція, що діє за правилом: висловлювання not p є протилежним за змістом до висловлювання p.

Застосування таких операцій називається булевою алгеброю.
Логічні операції та висловлювання дозволяють складати більш складні висловлювання, наприклад:

Перевіряється чи хоча б одна змінна x або y більше нуля
>>> x > 0 or y > 0
у наступному висловлюванню - чи обидві змінні більше нуля
>>> x > 0 and > 0

Записуючи складні висловлювання, як і у математиці, потрібно враховувати пріоритет виконання операцій.

Таблиця 4.2 Пріоритет операцій

Рівень Операції Назва операції
1 ** степінь числа
2 - від'ємне число
3 *, /, //, % множення, ділення, цілочисельне ділення, залишок
4 +, - додавання, віднімання
5 ==, !=, >, <, >=, <= порівняння
6 not логічне заперечення
7 and кон'юнкція
8 or диз'юнкція
9 = присвоєння

Приклад щодо використання пріоритетів операцій:

>>> 10*3**2
90
>>> (10*3)**2
900

Слід зауважити, що висловлювання допускають проведення обчислень прямо у їх запису, наприклад:

birthday_year_User1 = 2001
birthday_year_User2 = 2003
all_third millennium = birthday_year_User1 - 2000 > 0 and birthday_year_User2 - 2000 > 0
# У даному випадку значення змінної буде True
birthday_year = int(input('Рік Вашого дня народження: '))
if 0 < birthday_year - 1991:
    print('Ви народилися до набуття незалежності України')
else:
    print('Ви народилися у незалежній Україні')

У зазначеному прикладі послідовність виконання операцій наступна

Рис. 4.1

Не рекомендується записувати у висловлювання складні обчислювання
Інтерпретатор читаючи висловлювання зліва на право при кон'юнкції (and) після отримання першого хибного значення припиняє читання висловлювання і повертає False

Висновки:

  • для реалізації логіки щодо аналізу значень змінних у ході виконання програми використовуються висловлювання;
  • висловлювання складаються із одного або більше тверджень;
  • для запису тверджень застосовують логічні оператори;
  • логічні оператори мають свою пріоритетність для зняття невизначеності при їх запису.

5. Умови та умовні оператори. Розгалуження

Теорія - 15 хв, практика - 0 хв

Код програм, які розглядали на попередніх заняттях були лінійними. У них інструкції коду виконувалися одна за одною та були обов'язковими до виконання. У реальних програмах, коли алгоритм передбачає аналіз даних, що обробляються, виникає необхідність написання умов за якими будуть виконуватися ті чи інші інструкції. В умовах використовуються висловлювання, які були розглянуті у першому питанні. Для цього серед інструкцій мови програмування Python є умовний оператор, який записується наступним чином:

# блок з інструкціями 1
if висловлювання:
    # блок з інструкціями 2
else:
    # блок з інструкціями 3
Умовний оператор if та його складова else у коді йдуть на одному рівні відступу (рівні інструкцій 1), а у них вкладені блоки з інструкціями 2 та 3 (відступ на 1 рівень нижче)

Даний вираз слід читати наступним чином:

Рис. 5.1

Такий спосіб виконання логіки програми називається розгалуженням. Тобто, в залежності від виконання певних умов код програми виконується за різними сценаріями.
Умовний оператор може бути не повним, використовується лише перша частина:

# Блок з інструкціями 1
if висловлювання:
    # Блок з інструкціями 2
# Блок з інструкціями 3

У даному прикладі блоки 1 та 3 виконуватимуться у будь-якому випадку, а блок 2 лише у випадку коли "висловлювання" приймає значення True.
Слід зауважити, що умовний оператор допускає висловлювання у яких проводяться математичні обчислення прямо у їх запису, наприклад:

birthday_year = int(input('Рік Вашого дня народження: '))
if 0 < birthday_year - 1991:
    print('Ви народилися до набуття незалежності України')
else:
    print('Ви народилися у незалежній Україні')

У зазначеному прикладі послідовність виконання операцій наступна

Рис. 5.2

Для більш складних умов, наприклад можливості більше двох станів об'єкту, умовний оператор можна використовувати із вкладеністю, наприклад:

randomNumber = input('Введіть будь-яке число: ')
if randomNumber > 0:
    print('Це число більше 0')
else: 
    if randomNumber == 0:
        print('Це число рівне 0')
    else:
        print('Це число менше 0')
Усі інструкції у вкладених блоках умовних операторів мають відступ

Для більш зручного використання умовного оператора для застосування множини умов передбачений наступний синтаксис:

ball = input('Введіть кількість отриманих балів: ')
if ball > 89:
    letter = 'A'
elif ball > 79:
    letter = 'B'
elif ball > 64:
    letter = 'C'
elif ball > 54:
    letter = 'D'
elif ball > 49:
    letter = 'E'
else:
    letter = 'FX'
print('Оцінка компетентності за шкалою ЄКТС:', letter)

Така конструкція називається множинне розгалуження.

Слід зазначити, що є ще одна конструкція запису умовного оператора, яка називається тернарний оператор. Вона є еквівалентом запису 

variable_A = 0
if висловлювання:
    variable_A = математичний_вираз_1
else:
    variable_A = математичний_вираз_2

і записується наступним чином

variable_A = математичний_вираз_1 if висловлювання else математичний_вираз_2

Дана конструкція працює наступним чином:

Рис. 5.3

Такий запис скорочує кількість рядків коду, та дозволяє отримати бажаний результат.
Дану конструкція повертає значення, тобто:

age = 19
print('повнолітній' if age > 18 else 'неповнолітній')
# значення у консолі 
# повнолітній

У записі висловлювань також можливо використовувати функції, які видають значення до якого можна застосовувати оператори порівняння. Прикладом такого застосування є перевірка типу змінної

value = 100
if type(value) == int:
    print('Тип даних int')
value = 92.5
if type(value) == int:
    print('Тип даних int')
elif type(value) == float:
    print('Тип даних float')

Висновки:

  • розглянуті умовні оператори дозволяють створювати розгалуження у програмі;
  • основою для побудови умов є висловлювання, які базуються на застосуванні операторів порівнянь. Висловлювання у свою чергу поєднуються логічними операціями.

6. Область видимості змінних у вкладених блоках умовних операторів

Теорія - 15 хв, практика - 0 хв

Умовні оператори мають вкладені блоки інструкцій. Кількість таких інструкцій у блоці не обмежена, а отже цілком імовірно, що у блоках будуть введені змінні для реалізації логіки програми. Розглянемо такий випадок

value = 100
if value > 50:
    temp = value / 4
    print('Значення temp у блоці 1 рівне: ', temp)
print('Значення temp на рівні основної програми: ', temp)

У консоль буде виведено наступне:

Значення temp у блоці 1 рівне:  25.0
Значення temp на рівні основної програми:  25.0

Тобто оголошена змінна у вкладеному блоці temp може використовуватися в основному блоці програми. Змінимо значення value на значення 30, і виконаємо ще раз код програми
У консоль буде виведено наступне:

Traceback (most recent call last):
  File "e:\work\projects\discipline_of_programming_technology\temp\test.py", line 6, in <module>
    print('Значення temp на рівні основної програми: ', temp)
                                                        ^^^^
NameError: name 'temp' is not defined

Як видно інтерпретатор видав помилку, оскільки висловлювання в умовному операторі прийняло значення False і вкладений блок не виконувався. А отже змінна temp не була оголошена.
Тому написання таким чином коду програми є поганою практикою. Для уникнення помилок змінну, яку необхідно застосувати у подальшому оголошують на рівні блоку де планують її використання. Для цього змінній присвоюють початкове значення (значення за замовчуванням), або значення None. Даний тип даних якраз і призначений для подібних ситуацій.

Висновки:

  • оголошення змінних у середині вкладеного блоку можливо для проведення розрахунків;
  • оголошення змінної у вкладеному блоці та використання її у вищих рівнях є хибним шляхом, який веде до потенційних помилок.
