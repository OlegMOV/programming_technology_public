НАВЧАЛЬНІ ПИТАННЯ

1. Створення функції

Теорія - 5 хв, практика - 20 хв

🎯 Для того щоб користуватися функцією необхідно її створити, іншими словами її необхідно визначити. Після створення / визначення її застосування називають - виклик функції.

Для визначення функції перед її іменем (назвою) ставиться службова інструкція def. Це дозволяє інтерпретатору врахувати, що наступна змінна буде посиланням на функцію.

Назви функцій - це назва змінної, тому тут необхідно дотримуватися правил назв змінних

🧠Практичне завдання 1

Розпочато проект по створенню програмного додатку, який дозволяє вести автоматизований облік особового складу, який проходить дистанційні курси з кібернетичної гігієни. Необхідно розробити функції, які формують обрамлення даних, що будуть виведені у консоль.

  • верхнє обрамлення "|-----------------|"
  • нижнє обрамлення "++++++++++++"
Назва функції, як правило, має починатися із дієслова та складатися із двох і більше слів. Хорошим стилем для назви функції є camelCase
Для тимчасового позначення блоку інструкцій, які плануються для заповнення у подальшому, використовують службову інструкцію pass. Дана інструкція нічого не виконує, а тільки забезпечує можливість імітації правильного синтаксису блоку операторів для циклічних, умовних операцій та функцій.
def addUserInfo():
    pass
    
def editUser():
    pass

Висновки:

  1. При розробці функцій важливо приділяти увагу її назві. Для цього потрібно застосовувати дієслово англійської мови.
  2. При написанні коду програми корисно використовувати інструкцію pass, що забезпечить дієздатність програми при не заповнених блоках коду операторів керуванням виконання програми.

2. Передача аргументів функції

Теорія - 0 хв, практика - 115 хв

🧠Практичне завдання 2

  1. Створити структуру даних для збереження особистих даних військовослужбовців.
  2. Розробити функції, що дозволяють:
    • здійснювати введення особистих даних військовослужбовців;
    • виводити передані їй дані, обрамлені у наступний формат: "|>>> data <<<|"

🧠Практичне завдання 3

У продовження практичного завдання 2, розробити функцію, яка перевіряє введені дані, перед записом до розробленої структури даних.

Функції можуть бути вкладені в інші функції

🧠Практичне завдання 4

Розробити функцію, вкладену у функцію по запису даних, що дозволяє генерувати число та час внесення даних. Передбачити у структурі даних збереження такого значення.

🧠Практичне завдання 5

Розробити функцію зі змінним числом аргументів. Аргументами обрати додаткові дані про військовослужбовця, які можуть бути не обов'язково присутніми.

3. Рефакторинг та документування коду програми

🧠Практичне завдання 6

Теорія - 0 хв, практика - 30 хв

  1. Здійснити рефакторинг коду програми.
  2. Здійснити документування коду програми.

Висновки:

  1. Після написання коду програми важливо провести рефакторинг. Оскільки написаний при першій ітерації код був націлений на забезпечення виконання поставлених завдань. Подальша робота з кодом має бути направлена на забезпечення його читабельності.
  2. При рефакторингу здійснюється декомпозиція коду.
  3. Для забезпечення ідентичності вихідному коду важливо дотримуватися однакових типів даних.
  4. Задокументований код дозволяє швидко зорієнтуватися за що відповідає фрагмент коду та приступити до його вдосконалення / відлагодження.
