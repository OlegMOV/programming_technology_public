2. Базові поняття у програмуванні

Алгоритм – набір інструкцій, які описують порядок дій виконавця, щоб досягти результату розв'язання задачі за скінченну кількість дій. Алгоритм потрібен для подальшого написання програмного коду, що забезпечить виконання необхідних дій щодо обробки даних.

Програмний код (текст програми) – набір інструкцій написаних мовою програмування у формі, що її може прочитати і модифікувати людина.

Комп'ютерна програма – запис алгоритму у формі, придатній для виконання комп'ютером.

Програмування – процес побудови комп'ютерних програм.  

Розробка програмного забезпечення – комплексний процес створення програмного забезпечення, що включає в себе проектування, програмування, тестування та підтримку програмного забезпечення.

Мова програмування – формальна знакова система призначена для опису алгоритмів та структур даних, засобами якої можна виражати алгоритми для виконання їх за допомогою комп'ютера. Вона визначає набір лексичних, синтаксичних і семантичних правил, що задають зовнішній вигляд програми і дії, які виконує комп'ютер під її управлінням. Отже мова програмування це штучна мова, що використовується для передачі команд комп'ютерам.  

Компілятор – комп'ютерна програма, що перетворює (компілює) вихідний код, написаний мовою програмування високого рівня, на семантично еквівалентний код на мові програмування низького рівня, який, необхідний для виконання програми комп'ютером.

Компіляція здійснюється для конкретного сімейства операційних систем (Windows, Linux, MacOS, Android)

Інтерпретатор – комп'ютерна програма, що здійснює:  

  • аналіз інструкцій коду програми;
  • обробку, перетворення програми у код на мові програмування низького рівня;
  • ініціює виконання інструкції.

Транспілятор – комп'ютерна програма, що перетворює вихідний код, написаний мовою програмування високого рівня, на семантично еквівалентний код на визначеній мові програмування.

Ідентифікатор – це ім’я деякої сутності (змінної, функції, класу, типу даних) в тексті програми для її позначення.
При виборі ідентифікаторів необхідно дотримуватися таких правил (Python):

  • Першим символом ідентифікатора може бути буква з алфавіту (символ ASCII в верхньому або нижньому регістрі, або символ Unicode) або символ підкреслення «_».
  • Інша частина ідентифікатора (всі символи крім першого) може складатися з букв (символи ASCII в верхньому або нижньому регістрі, а також символи Unicode), символу підкреслення _ або цифр (0-9).
  • Імена ідентифікаторів чутливі до регістру. Наприклад, myname і myName - це два різні ідентифікатори (зверніть увагу на регістр літер "n" та "N").
  • Ідентифікатор не може співпадати з ключовими (зарезервованими) словом інтерпретатора Python. Перелік ключових слів можна отримати, виконавши послідовно дві команди: import keyword та print(keyword.kwlist).

Змінна - посилання на область пам'яті комп'ютера, в якій зберігається деякі дані (посилання на деякий об'єкт).

Літерал (literal – константа) – постійне значення певного типу даних, записане у вихідному коді комп'ютерної програми. Ним може бути число чи рядок.

Мова Python відноситься до мов з неявною строгою динамічною типізацією.

Неявна типізація означає, що при оголошенні змінної її тип не вказується (при явній типізації тип змінної вказується обов'язково). Як приклад мовою C++ цілочислова змінна записується наступним чином:

// C++
int var1 = 48;

# python
var1 = 48

Строга типізація (англ. strong typing) не дає можливості проводити операції у виразах з даними різних несумісних типів.

Не строга типізація (англ. weakly typed) дає можливість виконати операції над несумісними типами даних і отримати деякий непередбачуваний результат. Тобто, у мовах з строгою типізацією ви не можете складати наприклад рядки і числа, необхідне приведення до одного типу.

Виконання програми забезпечується операційною системою, яка для цього виділяє ресурси:

  • оперативну пам'ять,
  • ресурс процесора,
  • доступ до пристроїв введення / виведення.

Написання програми це завдання розробника, який має у процесі розробки реалізувати мовою програмування розроблений алгоритм. При цьому потрібно виключити синтаксичні помилки та мінімізувати логічні.

Отже, мова програмування Python - це набір службових інструкцій та конструкцій з них, які дозволяють реалізувати розроблений алгоритм. Крім службових інструкцій важливим елементом є синтаксис їх запису.

3. Синтаксис. Типи даних

3.1 Синтаксис мови Python

Синтаксис мови є достатньо простим. Основні правила для написання коду програми наступні:

  1. Будь-яка програма це робота з даними, які позначаються відповідними символами. Для надання значень символу застосовують знак "=", який читається як присвоєння змінній значення:
  number = 1
  # У даному записі ми визначаємо, що на даний момент часу 
  # у цій програмі змінній "number" присвоєно значення 1

  1. Для пояснення окремих частин коду програми використовуються коментарі. Так у прикладі вище використано синтаксис для коментаря. Він позначається знаком решітка # після якого прописується текст, який не сприймається інтерпретатором як інструкції (ігнорується інтерпретатором).

  2. Кінець рядка є кінцем команди (прикінцеві символи непотрібні);

  name = 'Лекційне заняття'
Слід дотримуватися правила: ОДИН РЯДОК - ОДНА ІНСТРУКЦІЯ.

Але в окремих випадках допускається введення кількох інструкцій в один рядок, при цьому їх розділяють крапкою з комою (;)

# здійснюється присвоюxвання значень двом змінним
# та одна інструкція для виведення у консоль

name = 'Лекція'; number = 1; print(name, ' № ', number)

# Вивід у консоль
# Лекція  №  1

Деякі інструкції можуть записуватися у кілька рядків. Для цього необхідно їх розміщувати у дужках (круглі, квадратні, фігурні), наприклад:

if (var1 < 10 and var2 > 7 and
counter > 40):
    print('Слід завершити розрахунок')
  1. Як правило, код програми поєднується у блоки. Блок починається відповідною інструкцією, логіку якої реалізує зазначений блок. Для того щоб інтерпретатор розумів це, такі блоки відділяються однаковими відступами від початку рядка. Як правило ці відступи рівні 4 знакам пробілу (пов'язано з читабельністю коду). Тому складні інструкції, які містять у собі блок коду мають записуються за таким правилом:
if (var1 > 10):
  print('Змінна досягла значення 10')
else:
  var1 = var1 + 1
  print('Змінна не досягла значення 10')

var2 = var1 + 2024

У наведеному вище коді, основна інструкція (if), після неї двокрапка та вкладені інструкції. Після чого знову ідуть інструкції коду програми.

3.2 Назви змінних

Правильний вибір імені змінної – одна з найважливіших і складних речей в програмуванні, яка відрізняє початківця від гуру.

Більшість часу ми витрачаємо не на написання коду, а на його розвиток.

Можливо, ці слова не очевидні, якщо ви поки що нічого великого не писали або пишете код “тільки для читання” (написав 5 рядків, віддав замовнику і забув). Але чим більш серйозні проекти ви будете робити, тим більш актуальні вони будуть для вас.

Що таке “розвиток”? Це коли я вчора написав код, а сьогодні (або через тиждень) приходжу і хочу його поміняти. Наприклад, вивести повідомлення не так, а отак … Обробити товари по-іншому, додати функціонал … А де у мене там повідомлення зберігається? А де товар? ..

Набагато простіше знайти потрібні дані, якщо вони правильно позначені, тобто коли змінна названа правильно.

Правила іменування

  1. Ніякої транслітерації. Тільки англійська

Чим погана транслітерація?

  • серед розробників усього світу прийнято використовувати англійську мову для імен змінних. І якщо ваш код потім потрапить до когось іншого, наприклад ви будете в команді більше ніж з однієї людини, то великий шанс, що транслітерація йому не сподобається.
  • українська транслітерація гірше читається і довше, ніж назви англійською.
  • у проектах ви напевно будете застосовувати бібліотеки, написані іншими людьми. Багато що вже готове, в розпорядженні сучасного розробника є маса інструментів, всі вони використовують назви змінних і функцій англійською мовою, і ви, звичайно, будете їх використовувати.
  1. Використовувати короткі імена тільки для змінних “місцевого значення”.

Називати змінні іменами, що не несуть смислового навантаження, наприклад a, e, p, mg – можна тільки в тому випадку, якщо вони використовуються в невеликому фрагменті коду та їх застосування очевидно.

  1. Змінні з декількох слів пишуться разом

Для таких випадків є кілька загальноприйнятих нотацій:

# Camel case
userName = 'Admin'
getData()
setUserName()
isEmpty

# Pascal case
MyComponent
CustomData

# Snake case
current_user
user_personal_data

# Screaming snake case
SESSION_ID
ACCENT_COLOR
  1. Ім’я змінної має максимально чітко відповідати даним на які вона посилається

3.3 Типи даних

Мова Python відноситься до мов з неявною строгою динамічною типізацією.  

  • неявна - при написанні коду не вказується тип даних для змінної;  
  • динамічна - тип даних встановлюється для змінної у момент читання рядку коду інтерпретатором.

Тип даних – це множина значень і операцій на цими значеннями.

В Python типи даних можна розділити на вбудовані в інтерпретатор (built-in) і не вбудовані, які можна використовувати при імпортуванні відповідних модулів.  
До основних вбудованих типів відносяться:

  1. None (невизначене значення змінної)
  2. Логічний тип ( Boolean Type )
  3. Числа ( Numeric Type )  
    a. int – ціле число  
    b. float – число з плаваючою точкою (дійсне число)  
    c. complex – комплексне число
  4. Послідовності ( Sequence Type )  
    a. list – список  
    b. tuple – кортеж  
    c. range – діапазон
  5. Рядки (Text Sequence Type )  
    a. str
  6. Бінарні послідовності ( Binary Sequence Types )  
    a. bytes – байти  
    b. bytearray – масиви байт  
    c. memoryview – спеціальні об'єкти для доступу до внутрішніх даних об'єкта через protocol buffer
  7. Множини ( Set Types )  
    a. set – множина  
    b. frozenset – незмінювана множина
  8. Словники ( Mapping Types )  
    a. dict – словник

Розглянемо другу частину визначення типу даних, а саме "...та операцій над ними". Це вказує на те, що для кожного типу даних, як сутності, передбачено в інтерпретаторі Python можливі операції над його значеннями. Тобто, при записі коду:

var1 = 4
var2 = 7
var3 = var1 + var2

print(var3)

інтерпретатор аналізуючи кожну інструкцію коду, фіксує, що:

  • змінній var1 присвоєно значення 4 і це ціле число,
  • змінній var2 присвоєно ціле число 7.
  • тому у третьому рядку знак + для інтерпретатора означає необхідність здійснення математичної операції додавання для цілих чисел розміщених за посиланням змінних var1, var2, з відповідним присвоєнням результату змінній var3.

Рис. 3.1

Крім операцій над змінними в інтерпретаторі для вбудованих типів передбачено вбудовані функції, які виконують поширені трансформації зі змінними відповідного типу.

Так для числових значень змінних

var1 = -7
abs(var1) # абсолютне значення числа
pow(var1, 3) # повертає var1 у степені 3

var2 = 5
max(var1, var2) # повертає більше із двох значень

var3 = '2.5'
var4 = float(var3) # перетворює строковий тип даних у числового дійсного значення

А також є багато інших вбудованих функцій, які часто використовуються при написанні коду програми.

Крім того, для розширення такої функціональності є додаткові бібліотеки, прикладом цього є бібліотека Math.  

Розглядаючи можливості Python щодо роботи з числовими даними слід зазначити, що для чисел доступні наступні операції, які позначаються відповідними операндами:

Символ операції Призначення Синтаксис
+ Додавання x + y
- Віднімання  x – y
* Множення x * y
/ Ділення x / y
// Знаходження цілої частини від цілочисельного ділення x // y,
8 // 3 результат 2
% Знаходження залишку від цілочисельного ділення x % y,
8 % 3 результат 2
** Піднесення до степеня x**y
Допускається використання в одному математичному виразі цілих (int) та дійсних (float) чисел
При діленні цілих чисел результат матиме дійсний тип float
Передбачено можливість скорочення записів, наприклад:

  a = a + b еквівалентно a += b

Для строкового типу даних, символи математичних операцій мають дещо інше значення, наприклад:

str1 = 'Hello, '
str2 = 'world!!!'
print(str1 + str2) # результат: Hello, world!!!

Тобто дві строкові змінні поєднуються в одну послідовність.

Маючи знання основних математичних операцій вже є можливість розробити першу програму.

4. Лінійна програма

Лінійна програма - програма де використовуються лише операції з типами даних та інструкції вводу / виводу їх значень.

Використовуючи інтерпретатор Python ми маємо доступ до консолі куди ми можемо виводити результати обчислень, використовуючи інструкцію print(), а також для забезпечення інтерактивності інструкцію input(), яка очікує введення даних від користувача. Наприклад:

print('Введіть своє ім\'я')
name = input()
print('Введено: ', name)

Для введення навіть такої програми є незручності, тому для написання програми використовують окремі файли, яким дають розширення *.py  

Для виконання набраного коду в консолі операційної системи викликається файл, що запускає інтерпретатор Python та шлях до файлу з кодом програми.

> python3 file.py

Таким чином, це і є лінійна програма, яка дає запит користувачу на введення його імя, і отримавши - виводить його у консоль.

Для математичних операцій з числами діють правила математики, а саме щодо пріоритетності їх виконання:

a = 3
b = 4
c = 5
d = a + b * c

print(d)

 Порядок виконання математичних операцій наступний:
1. Множення b * c
2. Додавання a + результат попереднього кроку
3. Присвоювання змінній d значення двох математичних операцій.
4. Виведення значення d в консоль.

Для написання коду програми, яка передбачає інтерактивну взаємодію з користувачем доцільно використовувати вкладені інструкції, наприклад для отримання значення змінної дійсного типу, код програми виглядатиме наступним чином:

height = float(input("Введіть Ваш зріст в метрах: "))
weight = (height - 1)*1.15
print("Окремі експерти вважать, що Ваша вага має бути: ", weight)

Введення дійсних чисел також дозволено з використанням експоненціального запису, тобто:

earthRadius = 6.371e3

Для проведення математичних обчислень, як правило, використовують модуль math.
Даний модуль містить константи:

math.pi #  3.141592653589793
math.e  #  2.718281828459045

Тригонометричні функції:

math.cos(X)

Функції із логіки:

# порівнює числа X та Y на їх близькість із заданими значеннями відхилення. у даному випадку не більше 2%

math.isclose(X, Y, *, rel_tol=0.02, abs_tol=0.0)

Для використання даного модуля необхідно його підключити до поточного проєкту. Для цього, як правило на початку коду програми робиться відповідний запис, що дозволяє використання функцій з цього модуля.

import math

s = math.sin(20)
print('Синус числа: ', s)

Для розуміння як працює та чи інша інструкція з переліку вбудованих, або із модулів що підключаються, в інтерпретаторі передбачена інструкція help(), яка дозволяє вивести опис інструкції / функції

import math

help(math.cos)

Висновки. Розглянутий вище матеріал дозволяє розробляти комп'ютерні програми для проведення математичних обчислень будь-якої складності.
